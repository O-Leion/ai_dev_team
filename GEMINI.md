# Project Rules (GEMINI.md)

このドキュメントは、Gemini CLI および全ての AI エージェントが遵守すべき最優先の指示です。

## 1. プロジェクトの基本方針
- **言語:** 原則として **Python 3.12+** を使用する。
- **インフラ:** **オンプレミス**運用を前提とし、**Docker / Docker Compose** で環境を構築する。
- **パッケージ管理:** **`uv`** を標準ツールとして使用する。
- **開発手法:** **ハイブリッド・スクラム**を採用するが、クライアントの指定があれば柔軟に変更する。

## 2. コミュニケーション・ルール
- **窓口:** クライアント（ユーザー）とのやりとりは、必ず **PM (Project Manager)** または **PL (Project Leader)** のペルソナが担当する。
- **言語:** 出力と言語は日本語で行う。

## 3. 技術標準
- **静的解析:** `ruff` を使用し、型チェック（pyright/mypy）を行う。
- **型定義:** Python の `typing` を活用し、厳格な型チェックを行う。
- **テスト:** `pytest` を使用し、機能実装時には必ずテストコードを付与する。

## 4. チーム構成とドキュメント
- 全ての意思決定と役割定義は `docs/` 配下のドキュメント（`TEAM.md`, `GUIDELINE.md` 等）に基づき、これを尊重する。
- メンバー選定は `docs/team/SELECTION_GUIDE.md` に従って、プロジェクトごとに最適化する。
