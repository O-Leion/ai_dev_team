# 標準技術ガイドライン (Standard Tech Guideline)

本プロジェクトチームは、クライアントの保守性と開発効率を最大化するため、以下の技術スタックを標準として採用します。

## 1. プログラミング言語
- **メイン言語:** **Python 3.12+**
  - 理由: クライアントが内容を把握しやすく、AI・バックエンド開発において豊富なエコシステムを持つため。
- **型定義:** `typing` モジュールを積極的に活用し、静的解析（pyright/mypy）による品質担保を行う。

## 2. パッケージ管理・ビルドツール
- **uv:** 高速なパッケージ解決とプロジェクト管理のために `uv` を標準使用する。
- **プロジェクト構成:** `pyproject.toml` を中心とした標準的な構成を維持する。

## 3. インフラ・実行環境
- **オンプレミス優先:** クラウドサービスに依存せず、ローカルサーバーや専用機材での運用を前提とする。
- **Docker:** 実行環境の差異をなくすため、全てのアプリケーションは Docker コンテナ上で動作させる。
- **Docker Compose:** データベース（PostgreSQL/Redis等）や複数のサービスをオーケストレーションするために使用する。

## 4. 開発ツール
- **Linter/Formatter:** `ruff` を標準採用し、高速かつ統一されたコード品質を維持する。
- **テスト:** `pytest` を使用した自動テストを基本とする。
