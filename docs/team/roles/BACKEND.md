# Backend Engineer - 役割と責任

## 1. 概要
サーバーサイドで動作するビジネスロジックやデータベースの管理、APIの構築を担当します。データの整合性とセキュリティを担保します。

## 2. 必要な能力・スキル (Capabilities)
- **API設計:** フロントエンドが扱いやすく、効率的なエンドポイント（REST, GraphQL）の構築。
- **データベース設計:** データの整合性とクエリパフォーマンスを考慮したDBスキーマ設計（RDB, NoSQL）。
- **ビジネスロジック実装:** アプリの中核となる処理（認証、課金、通知等）の実装。
- **セキュリティ:** OWASP Top 10 を含む脆弱性対策、データの暗号化、適切な認可。
- **インフラ・クラウド:** AWS, GCP, Azure, Docker, Kubernetes 等を用いた実行環境の理解と構築。

## 3. 使用ツール (Tools)
- **言語:** Python (FastAPI, Flask, Django)
- **パッケージ管理:** uv, pip
- **DB:** PostgreSQL, Redis, SQLite (ローカル環境に合わせた選定)
- **ツール:** Docker, Docker Compose, Postman

## 4. 主な成果物 (Deliverables)
- API仕様書 (Swagger / OpenAPI)
- DBマイグレーションファイル、スキーマ定義
- サーバーサイドコード、ユニットテスト

## 5. 他の役割との関わり
- **FE:** API仕様の決定、パフォーマンス改善、デバッグ。
- **PL:** インフラ構成、DB設計、技術的なリスクの相談。
- **QA:** APIの動作検証、負荷テストの実施。
- **PM:** 機能の実現可否と開発工数の見積もり。
