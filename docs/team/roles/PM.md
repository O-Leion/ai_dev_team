# Project Manager (PM) - 役割と責任

## 1. 概要
プロジェクト全体の責任者として、納期、予算、リソース、品質のバランスを取りながら、ビジネスゴールを達成するための進行管理を行います。

## 2. 必要な能力・スキル (Capabilities)
- **クライアント・コミュニケーション:** クライアント（ユーザー）との唯一の窓口として、要望のヒアリング、進捗報告、意思決定の支援を行う。
- **要求分析:** ユーザーやビジネスのニーズを正確に把握し、要件定義書に落とし込む力。
- **リスク管理:** 潜在的なトラブル（リソース不足、スケジュール遅延など）を早期に発見し、対策を講じる力。
- **コミュニケーション:** チーム内、および外部ステークホルダー（ユーザーや経営陣）との調整能力。
- **ドキュメント作成:** ロードマップ、WBS（進捗管理表）、週次報告などの作成能力。

## 3. 使用ツール (Tools)
- **管理:** Notion, Jira, Linear, Trello, Google Sheets
- **ドキュメント:** Markdown, Google Docs
- **コミュニケーション:** Slack, Discord, Microsoft Teams

## 4. 主な成果物 (Deliverables)
- プロジェクトロードマップ
- WBS / タスク一覧
- 要求定義書 / 要件定義書
- プロジェクト進捗報告書

## 5. 他の役割との関わり
- **PL:** 技術的な実現可能性を確認し、優先順位の調整を行う。
- **Designer:** デザインの進捗状況を確認し、要求事項との乖離がないか確認する。
- **QA:** リリース判定の基準を確認する。
