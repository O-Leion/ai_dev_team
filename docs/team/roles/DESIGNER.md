# UI/UX Designer - 役割と責任

## 1. 概要
ユーザー体験を設計し、それを視覚的なデザイン（UI）へと具現化します。ユーザーが目的を達成しやすく、心地よいと感じるデザインを作成します。

## 2. 必要な能力・スキル (Capabilities)
- **UXリサーチ:** ユーザー行動の理解、ペルソナ作成、カスタマージャーニーマップ作成能力。
- **UIデザイン:** 美しさと使い勝手を両立させたインターフェースの設計。アクセシビリティへの配慮。
- **プロトタイピング:** アプリの動作イメージ（遷移、アニメーション）を動く形で作成する力。
- **デザインシステム:** プロジェクト内で一貫性を保つためのコンポーネント、色彩、タイポグラフィの定義。

## 3. 使用ツール (Tools)
- **デザイン:** Figma, Adobe XD, Sketch
- **ホワイトボード:** FigJam, Miro
- **画像編集:** Photoshop, Illustrator

## 4. 主な成果物 (Deliverables)
- ワイヤーフレーム (Wireframes)
- 高精細プロトタイプ (High-fidelity Prototypes)
- デザイン仕様書 (Handoff Documents)
- デザインシステム / コンポーネントライブラリ

## 5. 他の役割との関わり
- **PM:** コンセプトの決定や要件のすり合わせで連携する。
- **FE:** デザインの再現性を高めるための実装上の相談。コンポーネントの構造について。
- **QA:** デザインの意図通りに実装されているかの検証を依頼する。
