# QA / Tester - 役割と責任

## 1. 概要
プロダクトが要件を満たし、品質が担保されているかを確認します。バグの早期発見と修正後の再検証、自動テストの導入等を通じて、リリース後の安定性を向上させます。

## 2. 必要な能力・スキル (Capabilities)
- **テスト計画:** プロジェクトのスケジュールに応じたテスト範囲、項目の策定能力。
- **バグ分析:** 不具合の再現手順、影響範囲の特定、チケット起票。
- **テスト自動化:** E2Eテストやリグレッションテストの自動化（Cypress, Playwright, Selenium等）。
- **ユーザー視点の検証:** 開発者が気づかないようなエッジケースや、使い勝手の悪さを発見する力。

## 3. 使用ツール (Tools)
- **テスト管理:** Qase, Zephyr, TestRail
- **自動化:** Playwright, Cypress, Appium
- **モニタリング:** Sentry, Datadog

## 4. 主な成果物 (Deliverables)
- テスト計画書 (Test Plan)
- テストケース一覧 (Test Cases)
- 不具合報告書 (Bug Reports / Tickets)
- テスト結果報告書 (Test Summary Report)

## 5. 他の役割との関わり
- **PM:** リリース判定会議。プロジェクトの品質状況を報告する。
- **PL:** 重大な不具合の修正優先順位の決定。
- **Designer:** デザイン、操作感、UIの微細な乖離を確認する。
- **FE/BE:** 不具合の詳細情報の共有、修正確認。
    - バグの再現手順を正確に伝え、修正を支援する。
