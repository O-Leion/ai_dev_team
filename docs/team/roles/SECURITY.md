# Security Engineer - 役割と責任

## 1. 概要
プロジェクト全体のセキュリティを統括し、サイバー攻撃、データ漏洩、脆弱性からプロダクトとユーザーを保護します。設計からデプロイ、運用までの全フェーズでセキュリティを担保します。

## 2. 必要な能力・スキル (Capabilities)
- **脆弱性診断:** Webアプリやインフラの脆弱性（OWASP Top 10等）を特定し、対策を講じる能力。
- **セキュリティ設計:** 認証・認可フロー、暗号化通信、鍵管理、セキュアなネットワーク構成の設計能力。
- **インシデント対応:** 攻撃を受けた際の検知、隔離、復旧手順の策定。
- **法規制・コンプライアンス:** 個人情報保護法、GDPR、ISMS 等のコンプライアンス要件の理解と実装支援。

## 3. 使用ツール (Tools)
- **スキャン:** Burp Suite, OWASP ZAP, Snyk, Trivy
- **認証:** Auth0, AWS Cognito, Keycloak
- **監視:** AWS CloudTrail, GuardDuty, WAF
- **管理:** Vault (HashiCorp), AWS Secrets Manager

## 4. 主な成果物 (Deliverables)
- セキュリティ要件定義書
- 脆弱性診断レポート
- インシデント対応マニュアル (IR Plan)
- セキュア・コーディングガイドライン

## 5. 他の役割との関わり
- **PL:** システム設計の初期段階でセキュリティリスクの洗い出しを行う。
- **BE:** API 認証、データ暗号化、セキュアな DB アクセスの実装を支援。
- **DevOps:** CI/CD パイプラインへのセキュリティスキャン (DevSecOps) の統合。
- **QA:** ペネトレーションテスト（侵入テスト）の共同実施。
- **PM:** セキュリティリスクと開発コストの優先順位付けについて協議。
