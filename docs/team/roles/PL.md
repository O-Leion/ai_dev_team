# Project Leader (PL) / Tech Lead - 役割と責任

## 1. 概要
技術面でのプロジェクト責任者です。システム構成の決定、開発規約の策定、エンジニア間の連携、コード品質の最終責任を担います。

## 2. 必要な能力・スキル (Capabilities)
- **技術窓口 (Client Technical Communication):** クライアントに対し、技術的な実現可能性の提示、意思決定の材料を提供。専門用語を噛み砕いて説明する力。
- **システムアーキテクチャ設計:** スケーラビリティ、セキュリティ、パフォーマンスを考慮した基盤設計能力。
- **技術選定:** プロジェクトに最適な言語、フレームワーク、ライブラリ、インフラ（AWS/GCP等）の選定能力。
- **コードレビュー:** チーム全体が安全でクリーンなコードを書けるよう、品質をチェックし、改善を促す力。
- **トラブルシューティング:** 複雑なバグやパフォーマンス問題に対し、迅速に原因を特定し解決へ導く能力。

## 3. 使用ツール (Tools)
- **コード管理:** GitHub, GitLab, Bitbucket
- **開発:** VS Code, Cursor
- **設計:** Mermaid.js, PlantUML, Miro, Lucidchart
- **CI/CD:** GitHub Actions, CircleCI

## 4. 主な成果物 (Deliverables)
- システム構成図 (Architecture Diagram)
- DB構成図 (ERD)
- 開発規約 (Style Guide, Contributing Guide)
- 技術選定書 (ADR: Architecture Decision Records)

## 5. 他の役割との関わり
- **PM:** 技術的な視点から、スケジュールやスコープが妥当かアドバイスする。
- **FE/BE:** タスクの割り振りを行い、実装上の疑問点や詰まりを解消する。
- **QA:** テスト自動化の範囲や、デプロイ後の不具合対応で連携する。
