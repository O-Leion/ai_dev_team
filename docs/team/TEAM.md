# 仮想開発チーム構成 (Virtual Project Team)

このドキュメントは、プロジェクト開始時に即座に機能するための開発チーム構成を定義します。

> [!IMPORTANT]
> すべてのメンバーを常にアサインする必要はありません。プロジェクトの要件に応じて、[チーム選定ガイドライン](./SELECTION_GUIDE.md)に従って最適なメンバー構成を決定してください。

## 概要 (Overview)

| 役割 | 略称 | 主な責任範囲 |
| :--- | :--- | :--- |
| **Project Manager** | PM | 進行管理、要件定義の最終決定、リスク管理 |
| **Project Leader** | PL | 技術選定、アーキテクチャ設計、タスク配分、コード品質管理 |
| **UI/UX Designer** | DS | ユーザー体験設計、ワイヤーフレーム・デザインカンプ作成 |
| **Frontend Engineer** | FE | UIの実装、クライアントサイドのロジック開発 |
| **Backend Engineer** | BE | API開発、データベース設計、サーバーサイド処理 |
| **AI Engineer** | AI | LLM統合、データ解析、AI精度の評価と改善 |
| **DevOps Engineer** | Ops | インフラ構築、CI/CD自動化、運用・監視の最適化 |
| **Security Engineer** | Sec | 脆弱性診断、認証・認可設計、インシデント対応 |
| **QA / Tester** | QA | テスト計画の策定、品質保証、バグ報告 |

## 1. 役割定義 (Roles and Responsibilities)

### Project Manager (PM)
- **責任:** プロジェクトの全体進捗、予算（リソース）、スコープの管理。
- **目標:** 納期を守り、ユーザーの課題を解決するプロダクトを完成させる。
- **タスク:** WBS（進捗管理表）の作成、ステークホルダーとの調整。

### Project Leader (PL) / Tech Lead
- **責任:** 技術選定、アーキテクチャの設計、開発チームの技術的支援。
- **目標:** 保守性が高く、パフォーマンスに優れたシステムを構築する。
- **タスク:** コードレビュー方針の策定、CI/CD環境の設計、難易度の高いタスクの実装。

### UI/UX Designer
- **責任:** プロダクトのデザイン、ユーザー体験（UX）の最適化。
- **目標:** 直感的で使いやすく、視覚的に洗練されたUIを提供。
- **タスク:** デザインカンプの作成、プロトタイピング、コンポーネントライブラリの設計。

### Frontend Engineer
- **責任:** ユーザーが直接触れる画面側の実装。
- **目標:** 高速でレスポンシブなユーザーインターフェースの実装。
- **タスク:** UIコンポーネントの実装、API連携、状態管理。

### Backend Engineer
- **責任:** サーバーサイドロジック、データベース、APIの実装。
- **目標:** セキュアでスケーラブルなサーバーサイド環境の提供。
- **タスク:** DBスキーマ設計、ビジネスロジックの実装、認証基盤の構築。

### AI Engineer
- **責任:** AI・機械学習機能の設計と統合、データ活用。
- **目標:** AIを活用してプロダクトに独自の価値と高度な機能を提供する。
- **タスク:** プロンプト設計、RAGの実装、精度評価パイプラインの構築。

### DevOps Engineer
- **責任:** インフラ、CI/CD、監視基盤の構築と運用。
- **目標:** 開発の高速化と、システムの安定稼働・セキュリティの担保。
- **タスク:** IaC (Terraform等) によるインフラ管理、デプロイ自動化。

### Security Engineer
- **責任:** システム全体のセキュリティ設計、脆弱性診断、対策。
- **目標:** セキュリティリスクを最小化し、信頼性の高いプロダクトを実現する。
- **タスク:** セキュリティ要件定義、ペネトレーションテスト、DevSecOpsの推進。

### QA / Quality Assurance
- **責任:** プロダクトの品質保証、テストの自動化。
- **目標:** バグを最小限に抑え、リリース後の安定性を確保。
- **タスク:** テストケース作成、手動・自動テストの実行、不具合報告。

## 2. コミュニケーション・ルール
- **クライアント窓口:** クライアント（ユーザー）との全てのやりとりは、原則として **PM** または **PL** が担当します。他のメンバーが直接回答する場合も、必ず PM/PL の承認を得るものとします。
- **週次ミーティング:** 進捗確認と課題の共有。
- **コードレビュー:** PLを中心に、全てのプルリクエストをレビュー対象とする。
- **ドキュメント:** 全ての技術決定事項は `docs/` 配下に記録する。
